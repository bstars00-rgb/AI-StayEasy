# StayEasy Vietnam — 모바일 앱 구상 (App Concept v0.1)

> 본 문서는 웹(현 프로토타입)에 이어 **모바일 앱**의 방향성을 정리한 기획 초안입니다.
> 포지셔닝·BM은 웹과 100% 동일하게 유지합니다: **OTA가 아닌 "호텔 직접예약 어시스턴트 & 콘텐츠 플랫폼"**, 수수료 0.

---

## 1. 한 줄 정의

> **여행자가 베트남 호텔을 더 잘 이해하고, 호텔 공식 채널에서 직접 예약하도록 돕는 모바일 여행 동반자 앱.**

앱은 웹의 "이해 → 직접예약" 흐름을, **여행 전·중 항상 손안에서** 쓰도록 확장합니다.

---

## 2. 왜 앱인가 (웹 대비)

| 웹이 잘하는 것 | 앱이 추가로 잘하는 것 |
|---|---|
| 발견·비교·SEO 유입 | **여행 중** 현장 활용(오프라인 가이드, 지도, 체크리스트) |
| 콘텐츠 소비 | **푸시**(체크인 D-day, 공식예약 혜택 만료 알림 — 비-스팸) |
| 1회성 방문 | **저장·위시리스트·여정** 등 개인화 재방문 |
| 광범위 접근 | **AI 추천**을 대화형으로, 카메라/위치와 결합 |

핵심: 앱은 **"여행 준비 → 현장 → 회고"** 전 주기를 잡아 리텐션과 직접예약 전환을 끌어올립니다.

## 4. 차별점 (Why StayEasy app)

1. **수수료 0 / 직접예약** — OTA 대비 공식 혜택을 명확히 노출
2. **AI 자연어 추천** — "키즈풀 있는 해변 가족호텔, 한국어 가능" → 매칭률+이유
3. **5개 언어 완전 현지화** — 한·영·베·중·일 (UI + 호텔 콘텐츠)
4. **신뢰 우선** — 스폰서 라벨 투명, 최저가 경쟁이 아닌 "적합성" 큐레이션
5. **현장 유틸리티** — 오프라인 가이드, 예약 전 체크리스트, 지도

## 5. 핵심 기능 (MVP → 확장)

### MVP (P0)
- **AI 검색**(온디바이스 추천, 5개 언어) — 웹 엔진 그대로 이식
- **목적지·호텔 탐색** + 필터(지역/여행유형/시설/공식혜택)
- **호텔 상세** — StayEasy 요약, 공식예약 혜택, 룸/위치/시설/취소 체크리스트
- **"Book on Official Website" 딥링크**(인앱 브라우저로 공식사이트)
- **위시리스트(저장)** + 최근 본 호텔
- **언어 전환 / 다크모드**
- **직접예약 가이드** 콘텐츠

### P1
- **여정(Trip) 보드** — 날짜별 후보 호텔·메모·체크리스트 모음
- **푸시 알림**(옵트인) — 체크인 D-3 준비 체크리스트, 위시리스트 혜택 변동
- **지도 뷰** — 호텔 위치/해변·공항 거리 시각화
- **오프라인 모드** — 저장한 호텔·가이드 캐시
- **공유** — 호텔/여정 링크 공유(딥링크)

### P2
- **AI → 실 LLM**(서버리스 프록시) — 대화형 여행 플래너("3박4일 가족 다낭 일정 짜줘")
- **리뷰/평점**(검증된 콘텐츠) · **환율·날씨·비자** 위젯
- **파트너 셀프서브 연동**(백오피스와 연결)

## 6. 정보구조 & 내비게이션

**하단 탭 5개 (Bottom Tab)**

1. **홈** — AI 검색 진입, 추천 목적지/호텔, 시즌 캠페인
2. **탐색(Explore)** — 도시 → 호텔 리스트·필터·지도
3. **AI 추천** — 자연어 입력 중심 전용 화면(웹 `/search` 확장)
4. **여정(Trips)** — 위시리스트·여정 보드·체크리스트
5. **더보기** — 가이드, 언어/테마, 파트너 문의, 정보

```
홈
 └─ AI 검색 결과 ─ 호텔 상세 ─ [공식사이트 딥링크]
탐색 ─ 도시 ─ (리스트 | 지도) ─ 호텔 상세
여정 ─ 여정 상세 ─ 체크리스트
더보기 ─ 직접예약 가이드 / 설정 / 파트너
```

---

## 7. AI 전략

- **MVP**: 웹의 온디바이스 추천 엔진(`searchEngine.ts`) 재사용 — 오프라인 동작, 키 불필요, 5개 언어 의도 인식.
- **P2**: **서버리스 프록시(Vercel/Cloudflare Functions)** 뒤에 **Claude API** 연결 → 대화형 플래너·자유질의. **API 키는 서버 보관**(앱에 미노출). 온디바이스 결과를 컨텍스트로 넘겨 품질↑/비용↓.

---

## 8. 알림 전략 (신뢰 유지 = 비-스팸)

- 기본 **옵트인**, 빈도 낮게.
- 허용 유형: ① 체크인 D-day 준비 체크리스트 ② 위시리스트 호텔의 공식예약 혜택 변동 ③ 저장한 도시의 시즌 정보.
- 금지: 무차별 프로모션, 최저가 압박(포지셔닝 위배).

## 9. 비즈니스 모델 (웹과 동일 — 수수료 X)

무료 등록 · 유료 공식예약 가이드 페이지 · 프리미엄 콘텐츠 · 추천 상단 노출(스폰서, 항상 라벨) · 공식예약 클릭 광고(CPC) · 시즌 캠페인.
→ 앱은 **CPC(공식사이트 클릭)**·**스폰서 노출**·**시즌 캠페인 카드**가 자연스러운 인벤토리.

## 10. 기술 스택 권장

- **React Native + Expo (TypeScript)** — 웹과 **로직/타입/i18n/검색엔진/디자인 토큰 최대 재사용**.
- 상태: 가벼운 store(zustand) + React Query(서버 도입 시).
- 네비게이션: React Navigation(하단 탭).
- 지도: react-native-maps.
- 저장: MMKV/AsyncStorage(위시리스트·언어·캐시).
- 딥링크/유니버설 링크: 공식사이트 인앱브라우저(`expo-web-browser`).
- 분석: 이벤트(검색→상세→공식클릭) 퍼널.

### 웹 자산 재사용 맵
| 웹 모듈 | 앱 재사용 |
|---|---|
| `data/hotels.ts`, `destinations.ts` | 그대로(또는 추후 API) |
| `lib/searchEngine.ts` + `searchI18n.ts` | **그대로 이식** |
| `i18n/locales/*`, `hotelContent/*` | **그대로 이식** |
| 디자인 토큰(브랜드/액센트/폰트) | RN 테마로 매핑 |
| 타입(`types/index.ts`) | 공유 패키지화 |

> 권장: `packages/core`(데이터·타입·i18n·검색)를 **웹/앱 공유 패키지**로 분리(모노레포).

## 12. MVP 범위 & 로드맵

- **Phase A (MVP, ~6주)**: 탭 5개, AI 검색, 탐색/상세/딥링크, 위시리스트, i18n/다크모드. (백엔드 없이 번들 데이터)
- **Phase B (~4주)**: 여정 보드, 지도, 푸시(옵트인), 오프라인 캐시.
- **Phase C (~6주)**: 백엔드 연동(콘텐츠/트래킹), AI 프록시(대화형 플래너), 리뷰/위젯.

## 13. 성공 지표 (KPI)

- **북커널**: 검색 → 호텔 상세 → **공식사이트 클릭(CPC)** 전환율

- 콘텐츠: 가이드 열람, 언어별 사용 비중
- 파트너: 스폰서 노출/클릭, 인콰이어리 수
